## 📊 支持的 CSV 格式

### ✅ 必须包含

| 字段 | 说明 | 示例 |
|------|------|------|
| **word** | 日文单词 | 食べる、猫、日本 |
| **reading** | 假名读音 | たべる、ねこ、にほん |
| **meaning** | 中文释义 | 吃、猫、日本 |

### ✅ 完整示例

```csv
word,reading,meaning
食べる,たべる,吃
飲む,のむ,喝
行く,いく,去
来る,くる,来
見る,みる,看
読む,よむ,读
書く,かく,写
聞く,きく,听
話す,はなす,说
歩く,あるく,走
```

### ✅ 支持的字符

- 日文汉字 (食、飲、行、来 等)
- 平假名 (ひらがな: あ、い、う)
- 片假名 (カタカナ: ア、イ、ウ)
- 中文释义 (简体/繁体)
- 空格和特殊字符

## ⚠️ 常见错误与解决

### ❌ 错误 1: 文件为空

```
❌ 文件为空，请选择有效的CSV文件
```

**原因**: 选择的文件没有内容  
**解决**: 检查 CSV 文件是否有数据

---

### ❌ 错误 2: CSV 格式错误

```
❌ CSV格式错误，请确保包含表头和至少一行数据
```

**原因**: 缺少表头或没有数据  
**解决**: 确保第一行是 `word,reading,meaning`

---

### ❌ 错误 3: 没有找到有效的数据行

```
❌ CSV中没有找到有效的数据行
```

**原因**: 所有数据行都缺少至少一个字段  
**解决**: 检查每一行都有三个信息（用逗号分隔）

---

### ❌ 错误 4: CSV 解析失败

```
❌ CSV解析失败，请检查文件格式
```

**原因**: 文件编码错误或格式问题  
**解决**: 保存为 **UTF-8** 编码的 CSV

## 🚨 重要提醒

⚠️ **在清除浏览器数据前**
- 导出学习数据备份
- 保存原始 CSV 文件
- 定期检查 localStorage 中的数据

⚠️ **文件编码**
- 必须使用 UTF-8 编码
- GBK 或 ANSI 会导致乱码
- 检查: 另存为 → 编码选双 UTF-8

⚠️ **分隔符**
- 必须用逗号 (`,`)
- 不能用分号 (`;`)、空格、制表符
- Excel 导出时特别注意

## ✨ 工作原理

```
用户选择 CSV 文件
    ↓
FileReader 读取 (UTF-8)
    ↓
字符串分行: split('\n')
    ↓
跳过表头: slice(1)
    ↓
逐行解析: split(',')
    ↓
验证数据: 检查三个字段
    ↓
转换格式: { id, word, kana, meaning }
    ↓
合并词汇: [...existing, ...imported]
    ↓
保存到 localStorage
    ↓
显示成功提示
    ↓
刷新 Feed 显示新词汇
```

A: 建议每次 20-50 个，循序渐进学习效果最好

**Q: 可以重复导入相同的词汇吗?**
A: 可以，但会创建重复。建议先检查再导入

**Q: 可以删除已导入的词汇吗?**
A: 当前版本不支持，可以清空 localStorage 重新开始

**Q: 导入的词汇来源?**
A: 可以来自课本、词典、其他 App、自己创建

**Q: 可以修改导入的词汇吗?**
A: 当前版本不支持，需要重新导入新 CSV
